# 시연 스크립트 — 관세포탈죄(저가신고 가격조작) · (주)로우텍무역 C-951

관세범죄 시뮬레이션(c1_price)을 **실제 포털 데이터로 구축**한 케이스의 화면별 실행 스크립트.
계정은 **임수사(본청 수사분야)** 로 로그인한 상태를 기준으로 한다.

## 사건 개요 (시연 도입 멘트)

> "(주)로우텍무역은 중국 선전에서 무선통신 모듈(HS 8517.62)을 수입하는 업체입니다.
> 동종물품 대비 40% 이상 낮은 단가로 24개월간 47건을 신고했고, 세관 제출용과 실거래용
> **이중 인보이스**를 사용하면서 차액을 **홍콩 HK NOMINEE LTD로 별도 송금**한 정황이 있습니다.
> 이 케이스로 관세수사 전 과정을 시연합니다."

| 항목 | 값 |
|---|---|
| 수사 대상 | (주)로우텍무역 (C-951) · 위험점수 87 (HIGH) |
| 혐의 | 관세수입 침해 — 관세포탈죄 + 가격조작죄 (관세법 §270①, §270의2) |
| 핵심 근거 | 신고단가 평균 USD 3.79 vs 벤치마크 6.70 (−43%) · 이중 인보이스 · 홍콩 차액송금 |
| 관련 인물 | 김포탈(대표, 지분 62%) · HK NOMINEE LTD(홍콩 정산법인) · (주)로우텍세컨드(가족법인) |

## 화면 1 — 관세수사 분석 > 진행중인 수사 : 사건 등록

**실행 스크립트**
1. 홈 하단 업무 바로가기에서 **관세수사** 클릭
2. `+ 수사 등록` 클릭
3. 수사대상 유형 **기업** → 수사대상 선택 **"(주)로우텍무역 (C-951)"**
4. 수사 유형 선택 **① 관세포탈 수사** → **등록**
5. 생성된 사건 카드에서 **혐의 지정**: 대분류 `① 관세수입 침해` → 죄명 `관세포탈죄` + `가격조작죄` 체크
   - 화면 하단에 "분석 관점: 정합성●·자금●·패턴●…" 과 **자동 구성 단계 미리보기**가 표시됨을 강조

**기대 결과 / 멘트 포인트**
- 혐의를 지정하는 순간 죄명별 분석 관점 매트릭스에 따라 분석 시나리오가 자동 구성됨
- "범죄 유형을 정의하면 시스템이 필요한 분석 단계를 스스로 설계합니다"

3. 우측 **분석 대상 기준 관계분석** 그래프:
   - `수입신고 관계분석` 뷰 — (주)로우텍무역 → 무선통신 모듈(8517.62) 수입신고 엣지,
     엣지 상세에서 출발항 CNSZX(선전)·해외거래처 SHENZHEN TELE CO·관세사 한빛관세사무소 확인
   - `위험구성 원인분석` 뷰 — 위험점수 87을 만드는 지표·신고 엣지(contributes)만 필터되어 표시
   - 노드 검색에 "SHENZHEN" 입력 → 공급자 중심 연결 강조

**기대 결과 / 멘트 포인트**

**실행 스크립트**
1. `기초자료 수집/등록` 탭 → **파일 업로드** 카드 클릭

3. 처리 AI 에이전트: 추천 유지(**수입신고검증 + 품목분류검증**) + **과세가격평가** 추가 선택
4. **업무특화RAG 분석서비스** 체크 → `신규 RAG 생성`
   - 이름 `C951 관세포탈 증거자료 RAG` · 검색 권한 `조사팀` · 유효기간 `6개월` → **파일 등록**
5. 하단 **외부 자료요청**: `금융거래` 카드에 "(주)로우텍무역 / 김포탈" 입력 → 자료요청 접수
6. **외부 정보 수집·정리**: 출처 `인터넷` / 제목 "무선모듈 시장단가 동향" / 메모에 시세 기사 URL → 추가

## 화면 4 — AI서비스 분석 작업 : 시나리오 실행

**실행 스크립트**
1. `AI서비스 분석 작업` 탭 — 혐의 지정으로 자동 구성된 단계 확인
   (미적용 시 분석 템플릿 `관세포탈 수사 템플릿` 선택 → **템플릿적용하기**)
   - 단계: 업무특화RAG 검색 → CDW 조회 → 관세정보 RAG → 조사정보 RAG → 수입신고검증 → 품목분류검증 → 과세가격평가 → 이상거래 검증 → 법령 검토 → 보고서 생성 → 보고서 검증
2. **수입신고검증** 단계 클릭 → 편집 패널 시연:
   - 입력값 스트립: `대상 기업/신고 = (주)로우텍무역 (C-951) · 워크스페이스 자동` / `첨부문서 추출값 = 기초자료 3건 자동 연결`
   - 자동 생성 프롬프트(패턴): "입력값 '분석대상 (주)로우텍무역 (C-951)'와 '등록 기초자료의 첨부문서 추출값(자동)'를 활용하여…" + [동작방식] 상세
   - **서비스 상세** 버튼 → 입력정의·벨리데이션·결과형식 팝업
3. **과세가격평가** 단계 → 분석범위 `저가신고 탐지` 체크 확인 → **이 AI서비스만 실행** (단일 실행 시연)
4. **단계별 자동실행** 클릭 → 실행 로그에서 단계 순차 진행
   - 완료 단계의 **결과 자동검증 배지**(형식 통과·필드 확인) 강조

**기대 결과 / 멘트 포인트**
- CDW 조회가 실제 DuckDB의 C-951 신고 47건을 집계해 응답
- 과세가격평가가 벤치마크 대비 −43% 편차와 추징 예상세액을 산출
- "각 단계의 프롬프트는 분석범위 선택에 따라 상세 동작방식까지 자동 구성됩니다"
